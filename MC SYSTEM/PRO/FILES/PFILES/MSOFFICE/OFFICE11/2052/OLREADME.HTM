使用 Outlook

安装 Outlook 后规则可能会不兼容

Outlook 包含下列新规则类型：

  • 用颜色标志标记某人发来的邮件。
  • 在新邮件通知窗口中显示某人发来的新电子邮件。

在 Outlook 2003 中使用这些新规则类型创建新规则后，如果使用 Microsoft Outlook 2002 Service Pack 1 或更早版本来访问该电子邮件帐户，这些规则可能无法正常工作。为了避免规则不兼容，请执行下列操作：

  • 卸载 Outlook 2003，然后安装 Microsoft Outlook 2002 Service Pack 2 或更高版本。
  • 在任一台用于访问同一帐户的计算机上安装 Microsoft Outlook 2002 Service Pack 2 或更高版本。

以前使用 Outlook 的加载项现在可能会提示您允许访问 Outlook 数据

Outlook 增强了 Outlook 对象模型的安全性，许多第三方软件公司使用这些对象模型将他们的产品与 Outlook 集成。这些新的安全性增强功能可能会提示您授予这些程序访问 Outlook 数据的权限。

在某些情况下，第三方软件公司提供的软件的更新版本可能不会提示您授予对 Outlook 数据的访问权限。有关此更改的技术信息，请参阅本文后面的 Outlook 对象模型一节和 Microsoft Developer Network (MSDN) 网站中标题为 Important Security Notes for Microsoft Outlook COM Add-In Developers 的文章。

ActiveX 控件不会加载到一次性窗体中

Outlook 接收到包含窗体定义的邮件时，该窗体被视为一次性窗体。为了防止不需要的脚本和控件在一次性窗体中运行，默认情况下，Outlook 不将 ActiveX 控件加载到一次性窗体中。管理员可以使用下列注册表设置之一来控制此行为：

HKCU\Software\Policies\Microsoft\Office\11.0\Outlook\Security

HKCU\Software\Microsoft\Office\11.0\Outlook\Security

AllowActiveXOneOffForms (DWORD)

不存在 - 仅加载下面列出的控件。（默认）
0 - 仅加载下面列出的控件。
1 - 仅允许安全控件。
2 - 允许所有 ActiveX 控件。

在一次性窗体中可以使用下列控件：

  • fm20.dll 中的控件
  • Microsoft Office Outlook Rich Format Control
  • Microsoft Office Outlook Recipient Control
  • Microsoft Office Outlook View Control

使用 Outlook 2003，然后使用任何 Collaboration Data Objects Library (CDO) 1.2x 应用程序可能引起 Microsoft Exchange Server 停止工作

如果您使用 Outlook 2003，然后使用任何 Collaboration Data Objects Library (CDO) 1.2x 应用程序如 Microsoft Outlook Web Access for Microsoft Exchange Server 5.5，并且如果您以前没有安装下面知识库 (KB) 文章中提到的修补程序，Exchange Server 可能停止工作。

注意  Exchange Server 5.5 以后的 Outlook Web Access for Exchange Server 版本不受影响，因为以后的版本不使用 CDO。但是，其他启用 CDO 的应用程序受影响，除非您为您使用的 Exchange Server 版本安装了修补程序。

有关此问题的详细信息，请参阅适用于您使用的 Exchange Server 版本的知识库文章 Microsoft Office Product Support for Office Users。

Exchange Server 2003 SP1    #823343 CDO Heap Corruption When You Try to Access an Exchange Mailbox

Exchange Server 2000    #824960 CDO Heap Corruption When You Try to Access an Exchange Mailbox

Exchange Server 5.5    #818709 Outlook Web Access Stops Responding When You Try to Access an Exchange Mailbox

Outlook 对象模型

对象模型中阻止的其他属性：Outlook 对象模型保护

除继承了 Outlook 2002 对象模型保护行为之外，出于安全考虑，Outlook 2003 还阻止试图访问 Outlook 对象模型特定属性的代码。这些附加的阻止行为可能会导致使用 Outlook 以前版本的一些程序受到安全性阻止。在 Outlook 2003 中被阻止的附加属性有：

  • ContactItem 对象的 IMAddress 和 Body 属性
  • MailItem 对象的 HTMLBody 和 Body 属性
  • PostItem 对象的 HTMLBody 和 Body 属性
  • AppointmentItem 对象的 Body 属性
  • TaskItem 对象的 Body 属性
  • TaskRequestItem 对象的 Body 属性
  • TaskRequestAcceptItem 对象的 Body 属性
  • TaskRequestDeclineItem 对象的 Body 属性
  • TaskRequestUpdateItem 对象的 Body 属性
  • DistListItem 对象的 Body 属性
  • JournalItem 对象的 Body 属性
  • MeetingItem 对象的 Body 属性
  • ReportItem 对象的 Body 属性
  • RemoteItem 对象的 Body 属性
  • NoteItem 对象的 Body 属性
  • DocumentItem 对象的 Body 属性
  • Inspector 对象的 HTMLEditor 和 WordEditor 属性
  • MailItem 对象的 HTMLBody 属性、Body 属性和 SenderEmailAddress 属性
  • PostItem 对象的 HTMLBody 属性、Body 属性和 SenderEmailAddress 属性
  • MeetingItem 对象的 Body 属性和 SenderEmailAddress 属性

使用会议工作区

将 Outlook 会议要求链接到会议工作区时的注意事项

非定期会议
如果以前尚未将工作区链接到定期会议序列，则可以将多个非定期会议要求链接到同一会议工作区。

定期会议

  • 只能将一个定期会议序列链接到一个工作区。将序列链接到工作区时，该工作区不能已链接到其他会议。
  • 不能将定期会议序列中的某次会议与主要序列分别链接到不同的工作区。
  • 一旦从定期会议序列的会议要求中删除了会议工作区链接，则以后不能再将该会议要求链接到同一工作区。但可以将其链接到其他工作区。
  • 使用 Outlook 更改现有会议序列的定期模式以减少重复次数时，并不会在工作区中实施对模式进行的所有更改。如果工作区中已经包含序列中某些日期的数据（如日程或文档），则不会删除这些日期。
  • 打开定期会议序列或约会序列的单个实例时，Outlook 中的会议工作区任务窗格不可用。

使用代理
若要允许 Outlook 代理为您创建会议工作区，该代理必须对将在其中创建会议工作区的 Microsoft Windows SharePoint Services 网站具有访问权限。

SharePoint 通知

在“规则和通知”对话框中查看 Internet 网站上的 SharePoint 通知

若要在 Outlook 中接收 Microsoft Windows SharePoint Services 通知，需要将 SharePoint 服务器地址添加到注册表中的下列位置：

HKEY_CURRENT_USER/Software/Microsoft/Office/11.0/Common/Security/Trusted Alert Sources "siteurl" (REG_SZ) = 1

其中，如果 SharePoint 网站为 http://www.fourthcoffee.com，则“siteurl”字符串应为“www.fourthcoffee.com”。
